mpf2k ideas!

list tracks in view (automatic TRUCKNUMBER assignment) by appearance of [[tracks]] groups, this way we can keep embedded TRACKNUMBER tag only for writing it to files and archival purposes.

also this idea allows us to create any metadata.toml file for any playlist in the future, standardizing format for all future entries.

-

add new [generate] section to config.toml where we can specify semantic clustering of tags when generating metadata.toml by {cluster_string} headers by this specific rules:

tags are specified in sort order separated by comma

cluster is generated only if at least 1 tag from list is presesnt

only existing tags are placed in each cluster

\n -> inject new line

* -> list all tags globbed in alphabetic order

examples of {cluster_string}s:

"" (main header) = ALBUM,ALBUMARTIST,DATE,GENRE

"# -- Release Group / Issue --" = COMMENT,MEDIA,COUNTRY,LABEL,CATALOGNUMBER,RELEASE_YYYY_MM

"# -- Source --" = SOURCE,MAGNET

"# -- Digital Identifiers --" = DISCOGS_URL,MUSICBRAINZ_URL,\n,ACCURIPID,CTDBID,DISCID,\n,MUSICBRAINZ_*

-

always-write-to-tracks = list of uppercase tags to always write to [[tracks]] section even if they are common for entire album
  e.g. always-write-to-tracks=TRACKNUMBER,ARTIST,TITLE
